Here is a clean MASTER PROMPT you can directly use for your ICU SaaS / EMR build spec.

⸻

🧠 MASTER PROMPT — ICU ORDER ENTRY SYSTEM (NO AUTO-LINKING LOGIC)

SYSTEM GOAL

We are building a clinical order entry system (CPOE-style ICU module) where the interface is designed for maximum clinician freedom and safety, without automatic medical inference or forced relationships.

The system must behave as a neutral structured order composer, not a decision-support engine.

⸻

🗄️ DATABASE DESIGN PRINCIPLE

We will create independent master tables only.

There will be NO hard-coded relationship tables between antibiotics, cultures, investigations, or devices.

All relationships are user-selected at runtime by the doctor.

⸻

📦 CORE TABLE STRUCTURE

We will build separate independent tables:

1. antibiotic_master

* Contains full list of antibiotics
* Each entry is independent
* No forced mapping to anything else

2. route_master

* Contains all possible drug routes
* Examples: IV, Oral, IM, SC, NG, Nebulization, etc.

3. investigation_master

* Contains all lab + imaging + bedside tests
* Fully independent dropdown list

4. culture_master

* Contains all microbiology culture types
* Fully searchable multi-select list

5. device_master

* Contains ICU devices (catheters, ventilators, lines, drains, etc.)
* Fully independent selection list

⸻

💊 ANTIBIOTIC MODULE BEHAVIOR

When a doctor selects an antibiotic:

1. Doctor selects ONE antibiotic from master list
2. System then shows:
    * Allowed routes for that antibiotic (from route_master mapping)
3. Doctor selects:
    * ONE OR MORE routes
4. Final selection becomes:

Antibiotic + Route(s)

✔ Example:

* Meropenem → IV Infusion
* Linezolid → IV + Oral

⸻

⚠️ IMPORTANT RULE

There is:

❌ NO automatic suggestion of:

* cultures
* investigations
* devices
* diagnoses

ONLY manual selection is allowed.

⸻

🔍 CULTURES MODULE

* Doctor can select multiple cultures at once
* Searchable dropdown / multi-select UI

Examples:

* Blood culture
* Urine culture
* BAL
* ETA
* CSF
* Wound culture

✔ No dependency on antibiotic or diagnosis

⸻

🧪 INVESTIGATIONS MODULE

* Fully independent multi-select list
* Can select multiple investigations together

Examples:

* CBC
* CRP
* Procalcitonin
* ABG
* LFT
* RFT
* Imaging (X-ray, CT, MRI)

✔ No linkage to antibiotic/device

⸻

🏥 DEVICE MODULE

* Doctor can select multiple devices
* Independent selection only

Examples:

* Foley catheter
* Central line
* ET tube
* Ventilator
* Arterial line

✔ No automatic linking

⸻

🧠 USER FLOW (IMPORTANT UX RULE)

Step 1: Select Antibiotic

→ choose from large searchable list

Step 2: Select Route(s)

→ allowed routes appear for that antibiotic

Step 3: Optional independent selections

Doctor can optionally add:

* Investigations (multi-select)
